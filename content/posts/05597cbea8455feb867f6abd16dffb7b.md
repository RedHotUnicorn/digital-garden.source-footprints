  Print Import Notion Data to Google Sheets
===================================

 Last Updated OnJanuary 8, 2024 byAna[< Knowledge Base](https://mixedanalytics.com/knowledge-base/)In this guide, we’ll walk through how to pull data from the Notion API directly into Google Sheets, using the [API Connector add-on](https://mixedanalytics.com/api-connector/) for Sheets. We'll first get an API key from Notion, and then set up a request to pull in data to your spreadsheet.

1. Log in to Notion and click **Settings & Members** on the left side navigation menu  
![notion-img1](https://mixedanalytics.com/blog/wp-content/uploads/notion-img1.jpg?x98178)
2. Click **Connections** and then **Develop or manage integrations**  
![notion-develop](https://mixedanalytics.com/blog/wp-content/uploads/notion-develop.jpg?x98178)![notion-develop](data:image/svg+xml,%3Csvg%20xmlns=%22http://www.w3.org/2000/svg%22%20viewBox=%220%200%201062%20549%22%3E%3C/svg%3E)
3. Click **+ New Integration**  
![notion-img3](https://mixedanalytics.com/blog/wp-content/uploads/notion-img3.jpg?x98178)![notion-img3](data:image/svg+xml,%3Csvg%20xmlns=%22http://www.w3.org/2000/svg%22%20viewBox=%220%200%20876%20627%22%3E%3C/svg%3E)
4. You'll be prompted to give your integration a name and a logo. These can be anything, but remember what you pick :p. Click **Submit**.  
![notion-img4](https://mixedanalytics.com/blog/wp-content/uploads/notion-img4.jpg?x98178)![notion-img4](data:image/svg+xml,%3Csvg%20xmlns=%22http://www.w3.org/2000/svg%22%20viewBox=%220%200%20721%20847%22%3E%3C/svg%3E)
5. You will be presented with an internal token that you can use to make your API requests.   
![notion-img5](https://mixedanalytics.com/blog/wp-content/uploads/notion-img5.jpg?x98178)![notion-img5](data:image/svg+xml,%3Csvg%20xmlns=%22http://www.w3.org/2000/svg%22%20viewBox=%220%200%20985%20341%22%3E%3C/svg%3E)
6. Scroll to the bottom of the page, make sure **Internal integration** is selected, and click Save. Congrats, you have your token! Copy it down as we'll use it shortly.  
![notion-img6](https://mixedanalytics.com/blog/wp-content/uploads/notion-img6.jpg?x98178)![notion-img6](data:image/svg+xml,%3Csvg%20xmlns=%22http://www.w3.org/2000/svg%22%20viewBox=%220%200%20651%20514%22%3E%3C/svg%3E)

Part 2: Allow Access for Your Token
-----------------------------------

By default, the token you've just created doesn't have access to your Notion data until you explicitly share access. To share access, you'll need to add it like this.

* Navigate to your page or database, and click the click the ••• button on top right.   
![notion-open](https://mixedanalytics.com/blog/wp-content/uploads/notion-open.jpg?x98178)![notion-open](data:image/svg+xml,%3Csvg%20xmlns=%22http://www.w3.org/2000/svg%22%20viewBox=%220%200%20336%20106%22%3E%3C/svg%3E)
* Scroll down to the bottom and click **+Add connections**
* In the resulting pop-up, search for and select the integration you created earlier.  
![notion-addconnection](https://mixedanalytics.com/blog/wp-content/uploads/notion-addconnection.jpg?x98178)![notion-addconnection](data:image/svg+xml,%3Csvg%20xmlns=%22http://www.w3.org/2000/svg%22%20viewBox=%220%200%20568%20638%22%3E%3C/svg%3E)
* You'll see a message informing you that your integration will now have access to the page. Click **Confirm**.

Now your page is available via the API.

Part 3: Example API Requests
----------------------------

Notion provides the following endpoints. Note that some of these requests use `GET` while others use `POST`, as this will be important for [pagination handling](#pagination).



| HTTP method | Endpoint |
| --- | --- |
| GET | [List all users](https://developers.notion.com/reference/get-users) |
| GET | [Retrieve block children](https://developers.notion.com/reference/get-block-children) |
| GET | [Retrieve a comment](https://developers.notion.com/reference/retrieve-a-comment) |
| GET | [Retrieve a page property item](https://developers.notion.com/reference/retrieve-a-page-property) |
| POST | [Query a database](https://developers.notion.com/reference/post-database-query) |
| POST | [Search](https://developers.notion.com/reference/post-search) |

### **Example - Query Records from a Notion Database**

This example shows how to pull records from a Notion database into Sheets.

1. First, get your Notion database ID. Get your Notion database ID from the URL (or [see below](#search) for another method). If your URL is `https://notion.so/xxxx?v=YYYYY`, `xxxx` is your database ID.  
![notion-dbid-url](https://mixedanalytics.com/blog/wp-content/uploads/notion-dbid-url.jpg?x98178)![notion-dbid-url](data:image/svg+xml,%3Csvg%20xmlns=%22http://www.w3.org/2000/svg%22%20viewBox=%220%200%20952%20437%22%3E%3C/svg%3E)
2. Open up Google Sheets and click **Extensions > API Connector > Open > Create request.**
3. In the request form, enter the following, substituting in your own database ID and API key:
	* **Application**: `Custom`
	* **Method**: `POST`
	* **Request URL**: `https://api.notion.com/v1/databases/*your_database_id*/query`
	* **Headers**
		+ `Authorization`: `Bearer *your_key*`
		+ `Notion-Version`: `2022-06-28`
		+ `Content-Type`: `application/json`
	* **Request body**: `{}`
4. Create a new tab and click **Set current** to use that tab as your data destination.
5. Name your request and click **Run**. A moment later you'll see database records populate your sheet.  
![notion-database](https://mixedanalytics.com/blog/wp-content/uploads/notion-database.jpg?x98178)![notion-database](data:image/svg+xml,%3Csvg%20xmlns=%22http://www.w3.org/2000/svg%22%20viewBox=%220%200%20834%20707%22%3E%3C/svg%3E)

### **Example - Search Available Notion Pages and Databases**

You can search all the databases you have access to with a request like this:

* **Application**: `Custom`
* **Method**: `POST`
* **Request URL**: `https://api.notion.com/v1/search`
* **Headers**
	+ `Authorization`: `Bearer` `*your_key*`
	+ `Notion-Version`: `2022-06-28`
	+ `Content-Type`: `application/json`
* **Request body**: `{"filter":{"value":"database","property":"object"}}`

To see available pages, change `database` to `pages` in the request body. To see all available entities, enter `{}` into the request body (but Notion [recommends](https://developers.notion.com/reference/post-search#limitations-of-search) being as specific as possible).

### **More Example API URLs**

For these, you'll need your page ID from the page URL  
![notion-img9](https://mixedanalytics.com/blog/wp-content/uploads/notion-img9.jpg?x98178)![notion-img9](data:image/svg+xml,%3Csvg%20xmlns=%22http://www.w3.org/2000/svg%22%20viewBox=%220%200%20727%20162%22%3E%3C/svg%3E)

Substitute in that page ID where it says `11111111`

1. **Get metadata about a page**   
`GET` `https://api.notion.com/v1/pages/11111111`
2. **Get content from a page**   
`GET` `https://api.notion.com/v1/blocks/11111111/children`

Part 4: Handle Pagination
-------------------------

By default, Notion API limits 100 responses per page as shown in their documentation:  
![notion-img13](https://mixedanalytics.com/blog/wp-content/uploads/notion-img13.jpg?x98178)![notion-img13](data:image/svg+xml,%3Csvg%20xmlns=%22http://www.w3.org/2000/svg%22%20viewBox=%220%200%201284%20547%22%3E%3C/svg%3E)

To get more records, use the `start_cursor` parameter as described. In API Connector you can do this automatically with [pagination handling](https://mixedanalytics.com/knowledge-base/pagination-handling/), like this.

* **Pagination type**: `cursor body`
* **Next token parameter**: `start_cursor`
* **Next token path**: `next_cursor` (this is the column name containing the tokens)
* **Run until**: *choose when to stop fetching data*  
![notion-pagination-cursorbody](https://mixedanalytics.com/blog/wp-content/uploads/notion-pagination-cursorbody.jpg?x98178)![notion-pagination-cursorbody](data:image/svg+xml,%3Csvg%20xmlns=%22http://www.w3.org/2000/svg%22%20viewBox=%220%200%20311%20239%22%3E%3C/svg%3E)

Note: the above example is for any of Notion's `POST` requests. If you're querying one of their GET endpoints, the pagination type will be `cursor` instead. The next token parameter and path will remain the same either way.Part 5: API Documentation
